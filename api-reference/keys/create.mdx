---
title: "POST /api/v1/keys — Create an API Key"
description: "Generate a new Splat API key with a descriptive name. The full s3d_ prefixed key is returned exactly once and cannot be retrieved again after creation."
---

Creating an API key gives you a credential you can use to authenticate requests to the Splat API. Each key is 36 characters long with a `s3d_` prefix and 128-bit cryptographic entropy — it is hashed server-side immediately after creation, so the full key value is **only returned in this response**. Your account supports up to 10 active keys at a time.

<Warning>
  The full API key is shown exactly once — in this response. Copy it to a secure location before closing or navigating away. If you lose it, you will need to revoke the key and create a new one.
</Warning>

<Note>
  This endpoint requires session authentication — you must be signed in to the [Splat dashboard](https://splat-3d.com/dashboard). You cannot use an existing API key to create a new key.
</Note>

## Endpoint

```
POST https://api.splat-3d.com/api/v1/keys
```

## Request

### Body parameters

<ParamField body="name" type="string" required>
  A descriptive name for the key. Use something that identifies its purpose or the environment it belongs to — for example, `"production-backend"` or `"ci-pipeline"`. This name appears in your key list and helps you track which key is used where.
</ParamField>

## Response

<ResponseField name="data" type="object" required>
  <Expandable title="properties" defaultOpen>
    <ResponseField name="id" type="string" required>
      The unique identifier for this key. Use this ID when revoking the key via `DELETE /api/v1/keys/{id}`.
    </ResponseField>
    <ResponseField name="name" type="string" required>
      The descriptive name you assigned to the key.
    </ResponseField>
    <ResponseField name="key" type="string" required>
      The full API key value, including the `s3d_` prefix. **This is the only time this value is returned.** Store it immediately.
    </ResponseField>
    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp of when the key was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object" required>
  <Expandable title="properties">
    <ResponseField name="request_id" type="string" required>
      A unique ID for this request. Include this when contacting support about a specific request.
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.splat-3d.com/api/v1/keys \
  --header "Content-Type: application/json" \
  --data '{ "name": "production-backend" }'
```

```python Python
import requests

response = requests.post(
    "https://api.splat-3d.com/api/v1/keys",
    json={"name": "production-backend"},
)
print(response.json())
```

```javascript Node.js
const response = await fetch("https://api.splat-3d.com/api/v1/keys", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ name: "production-backend" }),
});
const data = await response.json();
console.log(data);
```

</CodeGroup>

## Error responses

| Status | Code | Description |
|--------|------|-------------|
| `401` | `unauthorized` | You are not signed in via a valid dashboard session. |
| `422` | `limit_exceeded` | Your account already has 10 active API keys. Revoke an existing key before creating a new one. |

<Tip>
  Store your API key in an environment variable such as `SPLAT_API_KEY` and reference it at runtime. Never hardcode it in source code or commit it to version control.
</Tip>
