---
title: "GET /api/v1/keys — List Your API Keys"
description: "Retrieve metadata for all active API keys on your Splat account — id, name, created_at, and last_used_at. The full key value is never returned."
---

This endpoint returns all active API keys associated with your account. You can use the response to audit which keys exist, review when they were created, and identify keys that have not been used recently. Because Splat stores keys as SHA-256 hashes server-side, the full key value is never returned — only the `id`, `name`, `created_at`, and `last_used_at` fields are included.

## Endpoint

```
GET https://api.splat-3d.com/api/v1/keys
```

## Authentication

This endpoint requires a Bearer token in the `Authorization` header.

```
Authorization: Bearer s3d_your_key_here
```

## Response

<ResponseField name="data" type="object[]" required>
  An array of key objects. Each object represents one active API key on your account.

  <Expandable title="properties" defaultOpen>
    <ResponseField name="id" type="string" required>
      The unique identifier for the key. Use this ID to revoke the key via `DELETE /api/v1/keys/{id}`.
    </ResponseField>
    <ResponseField name="name" type="string" required>
      The descriptive name assigned to the key when it was created.
    </ResponseField>
    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp of when the key was created.
    </ResponseField>
    <ResponseField name="last_used_at" type="string">
      ISO 8601 timestamp of the most recent authenticated request made with this key. Returns `null` if the key has never been used.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object" required>
  <Expandable title="properties">
    <ResponseField name="request_id" type="string" required>
      A unique ID for this request. Include this when contacting support about a specific request.
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://api.splat-3d.com/api/v1/keys \
  --header "Authorization: Bearer $SPLAT_API_KEY"
```

```python Python
import requests

response = requests.get(
    "https://api.splat-3d.com/api/v1/keys",
    headers={"Authorization": f"Bearer {SPLAT_API_KEY}"},
)
print(response.json())
```

```javascript Node.js
const response = await fetch("https://api.splat-3d.com/api/v1/keys", {
  headers: { Authorization: `Bearer ${process.env.SPLAT_API_KEY}` },
});
const data = await response.json();
console.log(data);
```

</CodeGroup>

## Error responses

| Status | Code | Description |
|--------|------|-------------|
| `401` | `unauthorized` | Missing or invalid Bearer token. |

<Tip>
  Check `last_used_at` to spot keys that are no longer in use. If a key shows `null` or a stale timestamp, consider revoking it to reduce your attack surface and stay within the 10-key account limit.
</Tip>
