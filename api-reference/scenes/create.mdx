---
title: "POST /api/v1/scenes — Create a Scene"
description: "Create a new Splat scene and receive a presigned upload URL. Accepts video, photo zip, or R3D input with preset and parameter overrides."
---

Creating a scene is a two-step process. First, you call this endpoint with your file's content type and the processing options you want. The API registers the scene and returns a short-lived presigned `uploadUrl`. Second, you PUT your file directly to that URL — the file goes straight to cloud storage, bypassing the API entirely. Once the upload finishes, call [Start Processing](/api-reference/scenes/process) to kick off the reconstruction pipeline.

## Endpoint

```
POST https://api.splat-3d.com/api/v1/scenes
```

## Request

<ParamField body="title" type="string" required>
  A human-readable label for the scene. Displayed in the dashboard and returned in list and get responses.
</ParamField>

<ParamField body="content_type" type="string" required>
  MIME type of the file you're uploading. Must be one of:
  - `video/mp4` — a video walkthrough
  - `application/zip` — a zip archive of photos or an R3D capture
</ParamField>

<ParamField body="preset" type="string" default="standard">
  The quality preset to use. Controls resolution, training iterations, SfM settings, and output formats as a bundle. Options:
  - `fast` — 800px, 3K iterations, ~8 min, SOG only
  - `standard` — 1600px, 7K iterations, ~15 min, SOG
  - `quality` — 1600px, 15K iterations, ~25 min, SOG + PLY
  - `ultra` — full resolution, 30K iterations, ~45 min, SOG + PLY
  - `custom` — start from `standard` defaults and apply all your overrides

  See [Quality Presets](/guides/presets) for a full comparison.
</ParamField>

<ParamField body="sfm" type="object">
  Structure-from-motion settings. Overrides the preset defaults for SfM only.

  <Expandable title="sfm fields">
    <ParamField body="sfm.backend" type="string" default="glomap">
      The SfM engine used to estimate camera poses. Options: `glomap`, `colmap`, `none`. Use `none` only if you're supplying pre-computed poses.
    </ParamField>

    <ParamField body="sfm.matcher" type="string" default="exhaustive">
      How image pairs are matched during SfM. Options:
      - `exhaustive` — compares every pair; best for small/medium scenes
      - `sequential` — assumes frames are ordered; fastest for video
      - `vocab_tree` — uses visual similarity; best for large unordered photo sets
    </ParamField>

    <ParamField body="sfm.max_image_size" type="integer">
      Maximum image dimension in pixels before downscaling. Set to `0` for no downscaling. Accepted range: `0` or `400`–`4096`. Defaults vary by preset.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="training" type="object">
  Training configuration. Overrides the preset defaults for the Gaussian Splat training stage.

  <Expandable title="training fields">
    <ParamField body="training.iterations" type="integer">
      Number of training iterations. Range: `1000`–`50000`. More iterations generally produce higher quality but take longer. Diminishing returns typically kick in around 15K.
    </ParamField>

    <ParamField body="training.sh_degree" type="integer" default="3">
      Spherical harmonics degree for view-dependent color. Range: `0`–`4`. Higher values capture reflections and view-dependent effects but increase model size. `3` works well for most scenes.
    </ParamField>

    <ParamField body="training.densify_grad_thresh" type="float" default="0.0002">
      Gradient threshold for Gaussian densification. Range: `0.00001`–`0.01`. Lower values create more Gaussians (denser, higher detail); higher values create fewer (sparser, faster).
    </ParamField>

    <ParamField body="training.densify_interval" type="integer" default="100">
      How often (in iterations) the densification step runs. Range: `50`–`1000`. More frequent densification captures fine details earlier but may slow training.
    </ParamField>

    <ParamField body="training.opacity_reset_interval" type="integer" default="3000">
      How often (in iterations) Gaussian opacities are reset. Range: `500`–`10000`. Helps prune unnecessary Gaussians and prevent floaters.
    </ParamField>

    <ParamField body="training.resolution_scale" type="float" default="1.0">
      Scale factor applied to training images. Range: `0.25`–`2.0`. Values below `1.0` train at lower resolution (faster); above `1.0` trains at higher resolution (slower but captures more detail).
    </ParamField>

    <ParamField body="training.ssim_weight" type="float" default="0.2">
      Weight of the SSIM loss term relative to L1 loss. Range: `0.0`–`1.0`. Total loss is `(1 - ssim_weight) * L1 + ssim_weight * SSIM`. Higher values prioritize structural similarity.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="output" type="object">
  Output configuration. Controls which file formats are produced and whether quality metrics are computed.

  <Expandable title="output fields">
    <ParamField body="output.formats" type="string[]">
      Which output formats to produce. Options: `"sog"` (web-optimized, compressed) and `"ply"` (raw Gaussian Splat). Defaults vary by preset — `fast` and `standard` produce SOG only; `quality` and `ultra` produce both.
    </ParamField>

    <ParamField body="output.compute_metrics" type="boolean">
      Whether to compute PSNR quality metrics on a holdout set after training. Adds a small amount of processing time. Defaults vary by preset.
    </ParamField>

    <ParamField body="output.holdout_ratio" type="float" default="0.1">
      Fraction of images held out for metric evaluation. Range: `0.0`–`0.5`. Only relevant when `compute_metrics` is `true`. Set to `0` to use all images for training (no metrics will be computed).
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="data.sceneId" type="string">
  The unique identifier for the newly created scene. Use this ID in all subsequent requests for this scene.
</ResponseField>

<ResponseField name="data.uploadUrl" type="string">
  A presigned URL for uploading your file. PUT your file directly to this URL using the same content type you specified in the request. The URL is short-lived — upload within a few minutes of creating the scene.
</ResponseField>

<ResponseField name="meta.request_id" type="string">
  A unique ID for this API request. Include it when contacting support.
</ResponseField>

## Code examples

<CodeGroup>

```bash cURL
curl -X POST https://api.splat-3d.com/api/v1/scenes \
  -H "Authorization: Bearer $SPLAT_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "My First Scene",
    "content_type": "video/mp4",
    "preset": "standard"
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.splat-3d.com/api/v1/scenes",
    headers={"Authorization": f"Bearer {SPLAT_API_KEY}"},
    json={
        "title": "My First Scene",
        "content_type": "video/mp4",
        "preset": "standard",
    },
)
data = response.json()["data"]
scene_id = data["sceneId"]
upload_url = data["uploadUrl"]
```

```javascript JavaScript
const response = await fetch("https://api.splat-3d.com/api/v1/scenes", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${SPLAT_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    title: "My First Scene",
    content_type: "video/mp4",
    preset: "standard",
  }),
});
const { data } = await response.json();
const { sceneId, uploadUrl } = data;
```

## Step 2: Upload the file

After creating the scene, PUT your file directly to the presigned `uploadUrl`. Set the `Content-Type` header to the same value you passed in the create request.

```bash
curl -X PUT "$UPLOAD_URL" \
  -H "Content-Type: video/mp4" \
  --data-binary @walkthrough.mp4
```

<Tip>
  For best results, record a slow, steady walkthrough with good lighting. Avoid fast camera movements. 30–60 seconds of footage typically produces great results.
</Tip>

<Note>
  The upload goes directly to cloud storage — not through the Splat API. The 2 GB size limit applies to the file itself. There are no API-side size restrictions on the presigned upload.
</Note>

Once the upload completes, call [Start Processing](/api-reference/scenes/process) to begin reconstruction.
