Presets are the fastest way to control how Splat reconstructs your 3D scene. Each preset bundles a set of decisions — image resolution, training iterations, SfM settings, and output formats — into a single field. Instead of configuring every parameter individually, you pick the preset that fits your goal and Splat handles the rest. You can always override individual parameters on top of a preset when you need finer control.

## Preset comparison

| Preset | Resolution | Iterations | Est. time | Output | Best for |
|--------|-----------|------------|-----------|--------|----------|
| `fast` | 800 px | 3,000 | ~8 min | SOG only | Quick previews, testing |
| `standard` | 1,600 px | 7,000 | ~15 min | SOG | Production use, good balance |
| `quality` | 1,600 px | 15,000 | ~25 min | SOG + PLY | High-fidelity scenes |
| `ultra` | Full resolution | 30,000 | ~45 min | SOG + PLY | Maximum quality, archival |

<Info>
  Processing times are approximate. Actual duration depends on video length, the number of frames extracted, and current GPU availability.
</Info>

## Using a preset

Pass the `preset` field when creating a scene. If you omit the field, Splat defaults to `standard`.

```json
{
  "title": "Living Room Tour",
  "content_type": "video/mp4",
  "preset": "quality"
}
```

## Overriding individual parameters

Presets set sensible defaults, but you can override any individual parameter. Splat uses the preset as the base and applies your overrides on top. This lets you make targeted adjustments without manually specifying every option.

```json
{
  "title": "Custom Scene",
  "content_type": "video/mp4",
  "preset": "standard",
  "training": {
    "iterations": 10000
  }
}
```

This request uses all `standard` defaults — 1,600 px resolution, GLOMAP, SOG output — but bumps training iterations from 7,000 to 10,000.

## The `custom` preset

When you need full control over the pipeline, use `"preset": "custom"`. This starts from `standard` defaults and applies every override you provide. Use it when none of the named presets match your requirements.

```json
{
  "title": "Full Control",
  "content_type": "video/mp4",
  "preset": "custom",
  "sfm": {
    "backend": "colmap",
    "matcher": "sequential",
    "max_image_size": 2048
  },
  "training": {
    "iterations": 20000,
    "sh_degree": 4,
    "resolution_scale": 1.5
  },
  "output": {
    "formats": ["sog", "ply"],
    "compute_metrics": true,
    "holdout_ratio": 0.15
  }
}
```

## Preset details

<AccordionGroup>
  <Accordion title="fast — quick previews and testing">
    Use `fast` when you want results in under 10 minutes and visual fidelity is not the priority. Low resolution and few training iterations mean fine details will be lost, but the output is good enough to validate your capture workflow or prototype an integration.

    - **Resolution:** 800 px max dimension
    - **Training iterations:** 3,000
    - **Quality metrics:** Not computed
    - **Output formats:** SOG only
  </Accordion>

  <Accordion title="standard — balanced production default">
    `standard` is the default preset and a good fit for most production use cases. It strikes a practical balance between processing time and visual quality. The Splat web dashboard uses this preset.

    - **Resolution:** 1,600 px max dimension
    - **Training iterations:** 7,000
    - **Quality metrics:** PSNR computed with 10% holdout
    - **Output formats:** SOG
  </Accordion>

  <Accordion title="quality — high-fidelity customer-facing scenes">
    Use `quality` when visual fidelity matters. It runs at the same resolution as `standard` but trains for more than twice as long, producing sharper details and fewer artifacts. Both SOG and PLY are generated.

    - **Resolution:** 1,600 px max dimension
    - **Training iterations:** 15,000
    - **Quality metrics:** Computed
    - **Output formats:** SOG + PLY
  </Accordion>

  <Accordion title="ultra — maximum quality and archival">
    `ultra` uses full-resolution images (no downscaling) and 30,000 training iterations. It is designed for archival captures, research, or any use case where maximum fidelity outweighs processing time and file size.

    - **Resolution:** Full resolution (no downscaling)
    - **Training iterations:** 30,000
    - **Quality metrics:** Computed
    - **Output formats:** SOG + PLY

    <Warning>
      `ultra` processes significantly longer and produces larger files. Only use it when maximum quality is genuinely needed.
    </Warning>
  </Accordion>
</AccordionGroup>
